# Contributing

Thank you for considering contributing to Floating News Panel.

## How to contribute

1. Fork the repository.
2. Create a feature or fix branch.
3. Keep changes focused and documented.
4. Test the plugin in a Moodle development environment.
5. Open a pull request explaining what changed and why.

## Coding guidelines

- Follow Moodle coding style where possible.
- Keep strings in language files.
- Avoid hardcoded institutional names, colours or URLs.
- Do not include real user data, private screenshots or production images.
- Use Moodle APIs for forms, files, permissions and output.

## Testing checklist

Before opening a pull request, test:

- clean installation;
- adding a news item;
- editing a news item;
- deleting a news item;
- toggling active/inactive status;
- image upload;
- each layout mode;
- display on front page only;
- display site-wide;
- guest visibility setting;
- maximum active item limit;
- cache purge after upgrade.
